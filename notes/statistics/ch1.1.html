1.1. Sample Spaces

Probability theory is concerned with situations in which the outcomes occur randomly. Generically, such situations are called experiments, and the set of all possible outcomes is the sample space corresponding to an experiment. The sample space is denoted by \(\Omega\), and an element of \(\Omega\) is denoted by \(\omega\).

EXAMPLE A

Toss a coin \[\Omega = \{h, t\}\]

Roll a die \[\Omega = \{1, 2, 3, 4, 5, 6\}\]

Toss a coin twice \[\Omega = \{hh, ht, th, tt\}\]

EXAMPLE B

Driving to work, a commuter passes through a sequence of three intersections with trafﬁc lights. At each light, she either stops, \(s\), or continues, \(c\). The sample space is the set of all possible outcomes: \[\Omega = \{ccc, ccs, css, csc, sss, ssc, scc, scs\}\] where \(csc\), for example, denotes the outcome that the commuter continues through the ﬁrst light, stops at the second light, and continues through the third light.

EXAMPLE C

The number of jobs in a print queue of a mainframe computer may be modeled as random. Here the sample space can be taken as \[\Omega = \{0, 1, 2, 3, \dots\}\] that is, all the nonnegative integers. In practice, there is probably an upper limit, N , on how large the print queue can be, so instead the sample space might be deﬁned as \[\Omega = \{0, 1, 2, \dots, N\}\]

Events

An event is a subset of the sample space.

In Example B, the event that the commuter stops at the ﬁrst light is the subset of \(\Omega\) denoted by \[A = \{sss, ssc, scc, scs\}\]

In Example C, the event that there are fewer than ﬁve jobs in the print queue can be denoted by \[A = \{0, 1, 2, 3, 4\}\]

Union

The union of two events, \(A\) and \(B\), is the event \(A \cup B\) that either \(A\) occurs or \(B\) occurs or both occur.

From Example B, if \(A\) is the event that the commuter stops at the ﬁrst light, and if \(B\) is the event that she stops at the third light, \[A = \{sss, ssc, scc, scs\}\] \[B = \{sss, scs, ccs, css\}\] then \(A\cup B\) is the event that she stops at the ﬁrst light or stops at the third light and consists of the outcomes that are in \(A\) or in \(B\) or in both \[A \cup B = \{sss, ssc, scc, scs, ccs, css\}\]

Intersection

The intersection of two events, \(A\) and \(B\), is the event \(A \cap B\) that both \(A\) and \(B\) occur.

From Example B, if \(A\) is the event that the commuter stops at the ﬁrst light, and if \(B\) is the event that she stops at the third light, \[A = \{sss, ssc, scc, scs\}\] \[B = \{sss, scs, ccs, css\}\] then \(A\cap B\) is the event that she stops at the ﬁrst light or stops at the third light and consists of the outcomes that are in \(A\) or in \(B\) or in both \[A \cap B = \{sss, scs\}\]

Complement

The complement of an event, \(A^c\), is the event that \(A\) does not occur and thus consists of all those elements in the sample space that are not in \(A\) .

From Example B, the complement of the event that the commuter stops at the ﬁrst light is the event that she continues at the ﬁrst light: \[A^c = \{ccc, ccs, css, csc\}\]

Empty Set

The empty set, usually denoted by \(\emptyset\) , is the set with no elements. It is the event with no outcomes.

From Example B, if \(A\) is the event that the commuter stops at the ﬁrst light and \(C\) is the event that she continues through all three lights, \(C = \{ccc\}\), then \(A\) and \(C\) have no outcomes in common, and we can write \[A \cap C = \empty\] In such cases, \(A\) and \(C\) are said to be disjoint.

Laws From Set Theory

Commutative Laws: \[A \cup B = B \cup A\] \[A \cap B = B \cap A\] Associative Laws: \[(A \cup B\ ) \cup C = A \cup\ (B \cup C\ )\] \[(A \cap B\ ) \cap C = A \cap\ (B \cap C\ )\] Distributive Laws: \[(A \cup B\ ) \cap C = (A \cap C\ ) \cup\ (B \cap C\ )\] \[(A \cap B\ ) \cup C = (A \cup C\ ) \cap\ (B \cup C\ )\]
